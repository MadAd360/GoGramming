Help Guide



Code Repository:
-A repository is a storage location for code.
-In this application the users code is stored within a repository called "myRepo".
-Any other repositories the user has access to are called "<username>Repo".

Version Control:
-This application has git version control built-in. This allows users to share code in an effective manner.
-To track the changes in files the "Commit" option must be used and a message about the changes provided.
-To then share these changes with other users the "Push" option must be user.
-To get any changes made by other users the "Pull" option can be used.
-If there are any conflicts between changes made to the same file then whomever downloads the file must resolve these.

Editor Notes:
-To avoid issues with some compilation functionality please compile and run programs through the file with the "Main" method.
-To make the code editor fullscreen press "F11" when the code has been selected.
-When compiling code, several additional files (or the directories of the files) can be selected to be used if they are referenced in the code.

Accessing Code:
-The user code can be accessed from other applications using the repository address in the format below:

http://gogramming.co.uk:80/git/{{user}}
